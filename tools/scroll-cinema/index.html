ScrollCinema v2

A browser tool for turning image-frame sequences into cinematic, scroll-driven animations — no build tools, no dependencies, no server required. Open the file, drop your frames, click Export.

🍎
This is exactly how Apple does it. Visit apple.com/iphone in Chrome DevTools. You'll see a sticky canvas element, hundreds of PNG frames preloaded, and a scroll event mapping position to frame index. ScrollCinema generates that same pattern in minutes.
Getting Started

5-minute quickstart

No video yet? Start with demo frames to understand the tool, then replace with your real content.

1
Generate or import frames
Click 01 — Import tab. Either drop your MP4 video (frames extract automatically in your browser) or drag a folder of PNG/JPG files. No files? Click Generate Demo Frames to create 90 synthetic frames instantly.
⏱ Takes 5–30 seconds depending on frame count
2
Configure scroll speed and easing
Go to 02 — Configure. Set Scroll per frame — this is how many pixels of scroll advance the animation by one frame. 12px = fast, 40px = slow and cinematic. Pick an easing curve: Spring or Expo Out are closest to what Apple uses.
💡 Apple typically uses 20–30px/frame with ease-in-out
3
Add text layers (optional)
Go to 03 — Text Layers. Add a Headline that slides in at frame 20, and a Subtitle that fades in at frame 40. Drag the in/out sliders to place them exactly. Use Counter type to count up a number (like Apple's battery hours or camera megapixels).
✦ This is what makes it feel like a real Apple product page
4
Scrub and preview
Go to 04 — Preview. Drag the scrubber to scrub through the animation. Hit play to watch it at speed. Check your text layers appear and disappear at the right moments.
5
Export and drop into your project
Go to 05 — Export. Pick your format: HTML Snippet if you want a self-contained file, React for Next.js / Vite, Vue for Nuxt. Copy or download. Put your frames in a /frames/ folder and you're live.
📦 Vanilla JS · React · Vue 3 · Self-contained HTML
Concept

How scroll animations work

The technique uses a sticky canvas inside a tall scroll container. As the user scrolls, JavaScript maps the scroll position to a frame index and draws that image onto the canvas — 60 times per second.

// The core formula — everything else is decoration const scrollProgress = window.scrollY / totalScrollHeight // 0 → 1 const easedProgress = applyEasing(scrollProgress) // curved 0 → 1 const frameIndex = Math.round(easedProgress * (frameCount - 1)) canvas.getContext('2d').drawImage(images[frameIndex], 0, 0)

scrollPerFrame controls how many pixels of scroll = one frame. Lower = faster animation, fewer frames visible at once. Higher = slower, more deliberate, cinematic feel.

💡
Why canvas instead of img src swapping? Drawing to a canvas takes ~0.1ms. Changing an img src causes the browser to decode a new image each time, which causes jank at 60fps. Canvas is the correct approach for smooth frame-by-frame animation.

The sticky container pattern

<div style="position:relative; height:3000px"> <!-- tall scroll container --> <div style="position:sticky; top:0; height:100vh"> <!-- stays on screen --> <canvas id="sc"></canvas> <!-- draws each frame --> </div> </div>

The container height determines how much total scroll the animation gets. ScrollCinema calculates this as frameCount × scrollPerFrame.

Feature 01

Importing frames

ScrollCinema accepts images two ways: as individual files (PNG/JPG/WebP), or extracted from a video directly in your browser.

Importing image files

Drag a folder of images into the import zone, or click Choose Files. Files are sorted alphabetically — zero-padded names are essential:

Good naming — sorts correctly
frame001.png frame002.png frame003.png … frame120.png
Bad naming — sorts wrong (frame10 before frame2)
frame1.png frame2.png frame10.png frame11.png …

After loading, you'll see a frame strip, resolution, estimated memory usage, and duration at 24fps. Click any thumbnail to jump to that frame in the Preview tab.

⚠
Memory tip: 120 frames at 1920×1080 ≈ 950MB in RAM. For most use cases, 1280px wide and 60–90 frames is ideal. Use JPEG at 80% quality to reduce file sizes.
Feature 01 — Video

Video extraction (browser-side)

Drop an MP4 or WebM video and ScrollCinema extracts frames directly in your browser — no FFmpeg, no upload, nothing leaves your machine.

1
Drop your video
Drag an MP4, WebM, or MOV file onto the video drop zone. The extraction controls appear below.
2
Set extraction options
FPS: 24fps is standard. Use 12fps for lighter output, 30fps for ultra-smooth.
Max frames: Caps the total extracted. 120 frames is ideal for most use cases.
Width: Frames are resized to this width (maintaining aspect ratio). 1280px is recommended.
3
Click Extract Frames
A progress bar shows each frame being extracted. For a 5-second clip at 24fps, expect 120 frames in 10–20 seconds depending on your machine.
💡
Best source videos: Short clips (3–8 seconds), loopable product animations, 3D renders, or phone/laptop reveal animations. Export from Blender, After Effects, or Cinema 4D as MP4 H.264.

Feature 02 — Easing

Easing curves explained

Easing controls how the frame index maps to scroll progress. The visual curve previews in the Configure tab show exactly what each curve looks like.

CurveBehaviourBest for
linearConstant speed, 1:1 scroll-to-frameProduct rotations, data viz
ease-inStarts slow, acceleratesRevealing content that "arrives"
ease-outStarts fast, deceleratesElements settling into position
ease-in-outSlow → fast → slowGeneral purpose, most natural
expoNear-zero then explosive accelerationLaunch-style reveals
springSlight overshoot then settleOrganic, Apple-like feel ★
elasticOscillates past end before settlingPlayful, bouncy UI
bounceMultiple bounces at endHeavy objects landing
steppedJumps in 10% increments, no smoothingIntentional choppiness / glitch
🍎
Apple uses Spring. It provides a tiny overshoot that feels organic and intentional — like the animation has mass. This is the curve you'll see on apple.com product pages.
Feature 02 — Blend Modes

Blend modes

Blend modes apply a compositing operation to the canvas drawing, changing how frames are rendered visually — without touching your source images.

ModeEffectUse case
noneNormal drawing — no compositingDefault for all use cases
screenBrightens — dark areas become transparentLight leaks, glow effects, smoke on dark bg
multiplyDarkens — light areas become transparentTexture overlays on light backgrounds
overlayHigh contrast — darks darker, lights lighterDramatic cinematic look
color-dodgeExtreme brightening of highlightsSci-fi / neon glow aesthetics
differenceInverts based on difference between layersAbstract / glitch effects
💡
Blend modes are applied to the entire canvas. They work best on frames with black or white backgrounds. A 3D render on black with screen mode will make the background invisible, leaving only the object.
Feature 03

Text layers

Text layers are animated overlays that appear and disappear at specific frame positions — positioned over the canvas just like Apple's product copy.

Layer types

TypeWhat it does
HeadlineLarge italic serif text (Fraunces font). The main message per scene.
SubtitleSmaller supporting text, slightly transparent. Appears below the headline.
CounterA number that counts up from 0 to your target value as the user scrolls through the layer's frame range.
BadgeSmall uppercase label with accent background. For tags like "NEW" or "PRO".

Animation types

AnimationBehaviour
fadeFades in over the first 5% of its frame range, fades out over the last 5%
slide-upTranslates up 24px while fading in — classic Apple copy animation
slide-leftTranslates left while fading in
scaleScales from 85% to 100% while fading in
blurUnblurs from 12px to sharp while fading in — premium effect
countCounts from 0 to countTo value. Use with the Counter layer type.
Example: iPhone battery life reveal (Apple-style)

Three layers timed to reveal across an 80-frame sequence:

Layer 1 — Badge, "BATTERY LIFE", top-left, frames 0 → 80, fade Layer 2 — Counter, countTo: 29, center, frames 15 → 65, count Layer 3 — Headline, "hours", center, frames 20 → 65, slide-up Layer 4 — Subtitle, "All-day. Every day.", center, frames 30 → 65, fade

The number counts from 0 → 29 as the user scrolls, then the headline slides in beneath it.

Feature 03 — Counter

Count-up number layers

The Counter layer type counts a number from 0 to your target value as the user scrolls through its frame range. This is the exact mechanic Apple uses for specs like "29 hours battery", "48MP camera", "3nm chip".

How to set one up

  1. In the Layers tab, click + Counter
  2. Set the text to your number target (e.g. 29)
  3. Set animation to count
  4. Set Count to to your end value (e.g. 29)
  5. Set in/out frames to control when the counting happens
  6. Add a companion Headline layer with the unit (e.g. "hours") at the same position
💡
Place the counter's in-frame about 10 frames before the headline's in-frame. This way the number starts counting before the caption appears — the user sees the number build up, then the context word slides in. Extremely satisfying.
Complete camera spec example
Counter layer: text="48", animation="count", countTo=48, pos="center" in=10, out=60, size="clamp(56px,8vw,120px)" Headline layer: text="megapixel camera", animation="slide-up" in=20, out=60, pos="center", size="clamp(18px,2vw,28px)"
Feature 04

Preview & scrubbing

The preview tab shows your animation in a 16:9 canvas with a scrubber bar — drag it to any position to inspect the exact frame, check layer positions, and verify timing.

  • Scrubber: drag left/right to scrub through frames. Click anywhere on the track to jump.
  • Play button: plays the animation at the selected speed (0.25× to 2×).
  • Speed controls: 0.25× is useful for checking text layer fade-in timing precisely.
  • Frame counter: shows current frame and total frame count in the top right.
  • Text layers: your layers render live over the preview as you scrub — exactly as they'll appear on your page.
💡
After adjusting layer in/out frames in the Layers tab, come back to Preview and scrub slowly through that range to check the fade timing. The fade duration is 5% of your total frame count — for 120 frames that's 6 frames of fade in, 6 frames of fade out.
Feature 05

Exporting code

The Export tab generates production-ready code with your exact configuration baked in. All exports include progressive frame loading and reduced-motion support.

Progressive loading

All exported code loads frames in this order: frame 1 first (shown immediately), frame N last (preloaded), then all middle frames in sequence. This means the animation is visible and partially functional before all frames have loaded.

Progressive load order in exported code
// Load order: first → last → fill middle const order = [0, frameCount-1, ...Array.from({length: frameCount-2}, (_,i) => i+1)] order.forEach(i => { const img = new Image() img.src = basePath + 'frame' + pad(i+1) + '.png' img.onload = () => { images[i] = img } })

Reduced motion

All exports include a check for prefers-reduced-motion: reduce. If the user has enabled this accessibility setting, the scroll animation is disabled and the first frame is shown as a static image instead.

Feature 06

Live demo

The Live Demo tab renders your animation in a full-height scroll container — exactly as it will appear on your website. Scroll up and down to experience it as a real user would.

  • The HUD (top center) shows current frame, scroll progress %, and live FPS.
  • Text layers render exactly as configured in the Layers tab.
  • Blend modes are applied live.
  • Use this to verify the scroll speed feels right before exporting.
Integration

HTML snippet

The simplest integration — a self-contained snippet you paste into any HTML file. Put your frames in a /frames/ folder next to the HTML.

project/ ├── index.html ← paste the exported snippet here └── frames/ ├── frame001.png ├── frame002.png └── frame120.png

The snippet includes the wrapper div, sticky canvas, all JavaScript, and CSS — no external dependencies.

Integration

Vanilla JS module

An ES module class that you import and instantiate. Works in any project without a framework.

import { ScrollCinema } from './scroll-cinema.js' // Mount on a container element new ScrollCinema('#hero', { frameCount: 120, basePath: '/frames/', scrollPerFrame: 24, easing: 'spring', })

The container element needs a height (it's set to frameCount × scrollPerFrame automatically). Place it anywhere in your page.

Integration

React & Next.js

A React functional component using hooks. Works with Vite, CRA, and Next.js.

import ScrollCinema from './ScrollCinema' // In your page component: export default function HeroSection() { return ( <main> <ScrollCinema frameCount={120} basePath="/frames/" scrollPerFrame={24} easing="spring" onProgress={({ idx, progress }) => console.log(progress)} /> </main> ) }
⚠
Next.js App Router: Add 'use client' as the very first line of the component file. Scroll events require a browser environment and won't work in Server Components.

onProgress callback

The React component accepts an onProgress prop — a function called on every scroll event with {'{ idx, progress }'}. Use this to sync other elements with the scroll animation:

onProgress={({ progress }) => { // Sync a sidebar indicator, trigger analytics, etc. setScrollProgress(progress) }}
Integration

Vue 3 & Nuxt

A Vue 3 Single File Component using the Composition API. Works with Nuxt 3.

<!-- In your page or layout --> <template> <div> <ScrollCinema :frame-count="120" base-path="/frames/" :scroll-per-frame="24" easing="spring" @progress="onProgress" /> </div> </template> <script setup> import ScrollCinema from '~/components/ScrollCinema.vue' function onProgress({ idx, progress }) { console.log('Frame:', idx, 'Progress:', progress) } </script>
💡
Nuxt 3: Put the component in ~/components/ScrollCinema.vue and it auto-imports — no import statement needed in your page.

Performance tips

  • Use JPEG over PNG where possible — 80% quality JPEG is visually identical but 3–5× smaller. Your animation will load faster and use less RAM.
  • 1280px wide is the sweet spot — most viewport widths don't need 4K frames. 1280px at 80% JPEG quality per frame ≈ 40–80KB each.
  • 60–120 frames is enough — at 24fps that's 2.5–5 seconds of animation. More frames = more memory. Apple's iPhone page uses ~140 frames.
  • Use requestAnimationFrame — all exported ScrollCinema code wraps the scroll handler in rAF. Never bind heavy logic directly to the scroll event.
  • Passive scroll listener — all exported code uses {'{ passive: true }'} on the scroll event. This prevents jank on mobile.
  • Progressive loading — the exported code loads frame 1 and frame N first. The animation starts immediately. Don't show a loading spinner — let the user start scrolling.
  • Test on mobile — scroll animations on iOS Safari behave differently. Momentum scrolling means many frames can be skipped in a single tick. ScrollCinema handles this correctly with rAF, but always test on a real device.

Extracting frames with FFmpeg

If you prefer to extract frames before importing (rather than using the browser extraction), FFmpeg is the industry standard tool.

Basic extraction — 24fps, all frames
ffmpeg -i input.mp4 -vf fps=24 frames/frame%03d.png
JPEG output for smaller files (recommended)
ffmpeg -i input.mp4 -vf fps=24,scale=1280:-1 -q:v 3 frames/frame%03d.jpg
Limit to 120 frames maximum
ffmpeg -i input.mp4 -vf fps=24,scale=1280:-1 -frames:v 120 frames/frame%03d.jpg
Extract only a 5-second clip starting at 2s
ffmpeg -ss 2 -t 5 -i input.mp4 -vf fps=24,scale=1280:-1 frames/frame%03d.jpg

No FFmpeg? Use EZGIF.com/video-to-jpg — upload your video, set frame rate, download the ZIP, and import directly into ScrollCinema.

Troubleshooting

Canvas stays blank after loading frames

Usually a CORS issue when the exported file is opened via file://. Run a local server instead:

npx serve . # or python3 -m http.server 8080

Animation plays out of order

Frame files aren't zero-padded. Rename them with consistent 3-digit padding: frame001.png, not frame1.png.

Animation is choppy or drops frames

  • Wait for all frames to fully load before scrolling fast
  • Reduce frame resolution (1280px is enough)
  • Use JPEG instead of PNG
  • Test in Chrome — it has the best canvas performance

React: "window is not defined"

Add 'use client' as the first line of the component. Next.js App Router runs components server-side by default — scroll events need a browser.

Text layers don't appear

Check the in-frame value in the Layers tab. If inFrame is higher than your current preview position, the layer hasn't appeared yet. Scrub past that frame in the Preview tab.

Video extraction is very slow

Browser-based extraction is CPU-intensive. For very long videos (over 60 seconds), use FFmpeg directly and import the resulting images instead. Set Max Frames to 120 in the extraction options.

Mobile scrolling feels different

iOS Safari uses momentum scrolling. This means scroll events can fire at larger intervals, causing frames to be skipped. This is normal behaviour — the animation still plays correctly, it just may not show every single frame on fast swipes. To slow it down, increase scrollPerFrame.
