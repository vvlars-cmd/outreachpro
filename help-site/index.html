System Architecture
How OutreachPro is built
Docker-first Node.js. Single Alpine container. JSON data on Docker volume. No database. Server injection patches the frontend on every request — index.html is never touched directly.

Deployment
Docker Setup
# 1. Clone git clone https://github.com/vvlars-cmd/outreachpro && cd outreachpro # 2. Configure cp .env.example .env && nano .env # 3. Build and start docker compose up --build -d # 4. Verify curl http://localhost:3000/health # → {"status":"ok","ts":"...","port":"3000"} open http://localhost:3000/help # this docs site
Configuration
Environment Variables
# AI — personalisation, SDR, memory, forecast, proposals, voice scripts ANTHROPIC_API_KEY=sk-ant-api03-... # Scraping — Apify Google Maps, ICP builder, bulk scraper, competitor monitor APIFY_TOKEN=apify_api_... # Email — Gmail OAuth2 send, draft, inbox scraper GOOGLE_CLIENT_ID=....apps.googleusercontent.com GOOGLE_CLIENT_SECRET=GOCSPX-... GOOGLE_REDIRECT_URI=http://localhost:3000/api/gmail/callback # Optional PORT=3000 NODE_ENV=production
npm Package
CLI — npx outreachpro
npx outreachpro init # create .env npx outreachpro start # start :3000 npx outreachpro docker # docker compose up --build -d npx outreachpro help # show commands # npmjs.com/package/outreachpro
