Memory CRM — AI Follow-up Reminders

Memory CRM uses Claude to extract follow-up timing from reply emails and automatically queue re-engagement messages. Never forget a "call me back in Q4" again.

How It Works

  1. A lead replies: "We're all set for now, but check back in March"
  2. In Unibox, click Add to Memory on that reply
  3. Claude reads the message and extracts: follow-up date = March 1, intent = "maybe"
  4. A reminder is created and appears in CRM → Memory
  5. On March 1, the reminder shows as "Due" with a one-click re-engagement button

Timing Phrases Claude Understands

  • "Call me in Q4" → October 1
  • "After the holidays" → January 2
  • "Try me in 3 months" → today + 90 days
  • "Budget approved in March" → March 1
  • "We're reviewing vendors in spring" → April 1
  • "Not until next year" → January 1 next year

Due Memories

Go to CRM → Memory → Due Today to see all follow-ups that are due. Each entry shows the original message, the lead's name, and the extracted follow-up context. Click Re-engage to open the Email Composer pre-filled with a personalised re-engagement email that references the original conversation.

Manual Memory

You can also add memories manually without a reply text. Set the lead, follow-up date, and a note. Useful for adding reminders from phone calls or LinkedIn conversations that happened outside OutreachPro.

API

GET  /api/memory           → all memories
GET  /api/memory/due       → due today or overdue
POST /api/memory           → create {leadId, messageText, followUpDate?}
POST /api/memory/:id/action → mark actioned {note?}
DELETE /api/memory/:id     → delete
