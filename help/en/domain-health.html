Domain Health Dashboard

Before sending a single cold email, your sending domain needs to pass 5 technical checks. The Domain Health Dashboard runs all checks in seconds and gives you a score (A–F) so you know exactly what to fix.

Running a Check

  1. Go to Inbox Tester → Domain Health
  2. Enter your sending domain (e.g. yourbrand.com — not the full email address)
  3. Click Run Check
  4. Results appear within 3–5 seconds

The 5 Checks

  • MX Records — can your domain receive replies? Required for two-way email.
  • SPF — is your sending server authorised? Prevents spoofing.
  • DKIM — are your emails cryptographically signed? Proves authenticity.
  • DMARC — what happens when emails fail SPF/DKIM? Controls spam handling.
  • Spamhaus — is your domain on a global blacklist? Instant inbox death if yes.

Grading

  • A (90–100) — all checks pass. Safe to send campaigns.
  • B (70–89) — minor issues. Fix before sending at scale.
  • C (50–69) — significant issues. Do not send campaigns yet.
  • F (below 50) — critical failures. Fix immediately.

API

GET /api/domain-health/:domain
→ { domain, score, grade, checks: { spf, dkim, dmarc, mx, blacklist } }
