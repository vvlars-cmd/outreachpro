Multi-Sender Inbox Rotation

The safest daily send limit for cold outreach from a single inbox is 50 emails. With inbox rotation, you assign 3 warmed inboxes to a campaign and OutreachPro automatically spreads sends across all three — letting you safely send 150 emails per day while each inbox stays under its 50-email limit. Instantly.ai charges $97/month for this. In OutreachPro it's built in.

Why 50/Day Per Inbox?

Email providers (Gmail, Outlook) monitor sending volume from new addresses. Sending 150 cold emails in one day from a new inbox triggers spam filters. But the same 150 emails spread across 3 inboxes at 50 each looks completely normal — three people having normal email conversations.

Setting Up Rotation

  1. First, warm up at least 2–3 inboxes for 4–6 weeks (see Email Warmup guide)
  2. Go to Email Accounts → Inbox Rotation
  3. Enable rotation and add each warmed inbox
  4. Set daily limit per inbox (default: 50)
  5. Choose rotation strategy: Round-Robin, Random, or Weighted
  6. Assign the rotation pool to any campaign

Rotation Strategies

  • Round-Robin (recommended) — cycles evenly through inboxes in sequence. Inbox A sends email 1, B sends email 2, C sends email 3, A sends email 4... Most predictable distribution.
  • Random — picks a random available inbox for each send. Good for disguising patterns across large lists.
  • Weighted — assign weights to inboxes (e.g. Inbox A weight 2, B weight 1). Inbox A sends twice as many. Use this when one inbox has a stronger reputation.

Rotation Stats Dashboard

The rotation dashboard shows per-inbox: emails sent today, daily limit, remaining capacity, and total capacity across all inboxes. At a glance you can see if any inbox is near its limit and should be relieved.

What Leads See

Each lead receives email from whichever inbox was selected for their send. The sender name and domain rotate — but this is actually a benefit: if a lead receives a follow-up from a different sender address, it doesn't feel like a mass campaign. It feels like a different colleague reaching out.

API

GET  /api/rotation/config         → current rotation settings
POST /api/rotation/config         → update settings
POST /api/rotation/next-sender    → get next sender to use (auto-increments count)
GET  /api/rotation/stats          → per-inbox sent today / remaining
