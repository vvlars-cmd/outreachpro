Gmail Integration

OutreachPro connects to Gmail via OAuth2 to send emails, save drafts, and scrape your inbox for new leads — all without storing your password.

Setup (5 minutes)

  1. Go to Google Cloud Console → APIs & Services
  2. Create a new project (or use existing)
  3. Enable the Gmail API
  4. Create OAuth2 credentials: Web Application
  5. Add Authorized Redirect URI: http://localhost:3000/api/gmail/callback
  6. Copy Client ID and Client Secret to your .env file
  7. In OutreachPro: click Connect Gmail in Email Accounts
  8. Authorise in the popup — you're connected

Gmail Scraper

The Gmail Scraper scans your inbox for business emails and automatically extracts leads. Go to Gmail Scraper, set keywords (e.g. "Fahrrad", "Bike"), and click Run. OutreachPro streams matching emails and imports the sender's name, email, and company as leads.

Use this to capture inbound leads from people who've already emailed you — they convert at 10× the rate of cold outreach.

Sending & Drafts

  • Send directly — emails send immediately via Gmail API with rate limiting
  • Save as Draft — creates a Gmail draft you can review before sending
  • Batch Send — sends personalised emails to all leads in a campaign via SSE (streams progress live)

Troubleshooting

  • "Gmail not connected" — re-authorise by clicking Connect Gmail again
  • "Rate limit exceeded" — Gmail API allows ~100 emails/hour. OutreachPro handles throttling automatically
  • "Redirect URI mismatch" — make sure your Google Cloud OAuth settings include exactly http://localhost:3000/api/gmail/callback
