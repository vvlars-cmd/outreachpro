Workflow Automation

Workflows are rules that fire automatically when something happens. When a lead replies, their CRM deal moves to "Replied" automatically. When a buying signal hits 60+, their sequences stop automatically. No manual steps, no things falling through the cracks. Salesforce calls this "Flow" and includes it at $100+/user/month. It's built into OutreachPro.

Pre-Built Workflows

  • Reply received → move to Replied — CRM deal stage updates automatically when a lead replies to any email
  • Meeting booked → move to Meeting — scheduler booking auto-advances the CRM stage
  • Hot signal → stop sequences — when a lead scores 60+ on buying signals, all active sequences pause so you don't spam a hot lead
  • Proposal stage → generate proposal — (disabled by default) auto-triggers proposal generation when a deal reaches Proposal stage
  • Stale 14d → re-engagement — (disabled by default) queues a re-engagement campaign for deals with no activity in 14 days

Creating a Custom Workflow

  1. Go to Settings → Workflows
  2. Click New Workflow
  3. Set trigger: email_replied, meeting_booked, signal_score_60, stage_changed_proposal, or deal_stale_14d
  4. Set action: move_crm_stage, stop_sequences, generate_proposal, log_activity, or enqueue_reengagement
  5. Enable and save

Evaluating Workflows

Workflows are evaluated whenever OutreachPro processes an event. You can also trigger evaluation manually via the API for testing.

API

GET    /api/workflows              → list all workflows
POST   /api/workflows              → create {name, trigger, action, actionData}
POST   /api/workflows/:id/toggle   → enable/disable
POST   /api/workflows/evaluate     → {trigger, leadId?, dealId?} → run matching workflows
DELETE /api/workflows/:id          → delete
